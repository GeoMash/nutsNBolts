General Structure

Core Admin is a theme built with middleman. What this means is you get a lot of goodies that you can use to easily maintain the theme's structure in all aspects. You get by default the full power of SCSS, Compass, CoffeeScript, Sprockets, and much more!

The structure of the source folder looks like this:

  • font
  • images
  • javascripts
    • vendor - here are all the vendored javascript plugins
    • other files - things that are not in the vendor folder, which are theme specific (initializers etc)
  • stylesheets
    • bootstrap - all the bootstrap files in SCSS format, using assets from the bootstrap-sass gem
    • other files - any plugins or custom code goes here
  • layouts - contains the layout files and things that look the same on every page
  • pages - the actual pages that you'll find in this theme
  • partials - common pieces of html that define components and get rendered in multiple pages (easier to follow than going through the whole html page)

Stylesheet files

The structure of the stylesheets folder looks like this:

  • bootstrap - all the bootstrap files in SCSS format, using assets from the bootstrap-sass gem
  • application.css - the theme manifest in which all imports are declared
  • accordion.css.scss - restyled boostrap accordion plugin
  • action_nav.css.scss - the big buttons you see on top on the dashboard
  • _arrow.css.scss - custom mixin for generating arrow boxes (based on cssarrowplease)
  • badges.css.scss - restyled bootstrap badges and labels
  • base.css.scss - theme basic elements styles
  • bootstrap_alerts.css.scss - restyled bootstrap alerts
  • bootstrap-editable.css.scss - modified boostrap-editable css file - removed any background images except the loading spinner, added fontawesome characters instead
  • bootstrap-editable-fix.css.scss - custom code for the datepicker included in bootstrap-editable
  • bootstrap_include.css.scss - base file for importing the bootstrap library to get all of Bootstrap's styles, mixins and variables! (this is the file that the manifest actually imports, and NOT the boostrap folder itself)
  • bootstrap-wizard.css.scss - theme file for the wizard plugin
  • bootstrap-wysihtml5.css.scss - theme file for the boostrap wysihtml5 plugin
  • box.css.scss - code for the boxes you'll see in every page of the theme
  • breadcrumbs.css.scss - breadcrumbs and code for the right menu on the breadcrumbs bar
  • buttons.css.scss - all the goodies that make up the buttons - you can generate new button classes/colors by modifying the variables file
  • charts.css.scss - (dashboard) main chart size
  • chat-box.css.scss - defines the chat component which you can see in the widgets page
  • content_progress.css.scss - custom progress bars
  • dataTables.css.scss - data tables theme, changed to use different pagination styles, and fontawesome
  • dropdowns.css.scss - restyled bootstrap's dropdowns (contains dark/light colors)
  • easy_pie_chart.css.scss - easy pie chart, sizes and positioning
  • error_pages.css.scss - code for the 404 error page
  • font-awesome.css.scss
  • font-awesome-mixins.css.scss - custom mixins based on fontawesome, which can help include font characters easily anywhere, without html markup
  • form_elements.css.scss - all form components
  • fullcalendar.css.scss - full calendar plugin stylesheet
  • gritter.css.scss - jQuery Gritter (Growl) notifications plugin theme
  • ibutton.css.scss - remake of the iButton plugin from giva labs, to use css3 gradients and shadows
  • justgage.css.scss - justgage stylesheet for dimensions
  • message_box.css.scss - stylesheet for the (closable) message box widget
  • mixin_helpers.css.scss - a collection of mixins used throughout the theme
  • modal.css.scss - modal stylesheet
  • navigation.css.scss - custom mods for the top navigation/sub-navigation
  • numeric_stats.css.scss - stylesheet for the numeric statistics (used in sidebar, top-bar and widgets)
  • pagination.css.scss - restyled boostrap pagination
  • responsive480.css.scss - responsive code
  • responsive768.css.scss - responsive code
  • responsive979.css.scss - responsive code
  • responsive-tables.css.scss - Zurb's responsive tables sheet
  • scrollable.css.scss - stylesheet for scrollable boxes
  • select2.css.scss - select2 stylesheet - only removed the use of background images
  • select2_overrides.css.scss select2 overrides, changed color theme, and using fontawesome
  • sidebar.css.scss - stylesheet for sidebar code and some of it's components
  • sparkline_box.css.scss - stylesheet for custom sparkline charts
  • stats.css.scss - the statistics widget on the front page (before the main chart)
  • tables.css.scss - adds new optional class for styling tables
  • tags.css.scss - stylesheet for the tags plugin
  • tooltips.css.scss - sparkline tooltips
  • touchTouch.css.scss - touchTouch photo gallery stylesheet
  • triangle_button.css.scss - triangle buttons generator classes (see the buttons page)
  • ui_custom.css.scss - custom jquery ui stylesheet for sliders
  • uniform_default.css - uniform js default stylesheet
  • validation_engine.css.scss - validation engine customized stylesheet
  • variables.css.scss - list of variables used in the theme. you can resize some components and change various colors here
  • widgets.css.scss - custom news widgets stylesheet
  • xcharts.css.scss - xCharts default stylesheet

JavaScript files

The structure of the javascripts folder looks like this:

  • vendor
    • bootstrap - bootstrap js folder
    • boostrap_wysihtml5 - the boostrap wysihtml5 plugin
    • editable - boostrap-x-editable plugin files
    • fullcalendar - the full calendar plugin
    • ibutton - iButton plugin from giva labs
    • justgage - justgage plugin for making cool animated gauges
    • validation - jquery validation engine
    • wizard wizard plugin
    • bootstrap_dropdown_hover.js - bootstrap dropdown on hover plugin
    • bootstrap-editable.js - manifest file for bootstrap x-editable
    • bootstrap-wysihtml5.js - manifest file for bootstrap wysihtml5
    • d3.js - d3 v2 for the xCharts plugin
    • dataTables.js - jQuery dataTables plugin
    • html5shiv.js - support for ie6-8 html5 tags
    • ibutton.js - manifest file for the iButton plugin
    • jquery-182.js - jQuery 1.8.2
    • jquery.easy-pie-chart.coffee - jQuery easy pie charts
    • jquery_gritter.js - jQuery Gritter
    • jqueryui1102.js - jQuery UI 1.10.2
    • jquery_uniform.js - jQuery Uniform - currently used only for file input styling, feel free to remove if you don't need it
    • justgage.js - manifest file for justGage
    • responsive-tables.js - Zurb's responsive tables
    • select2.js - select2 jquery plugin
    • sparkline.js - jQuery Sparklines
    • tags_input.js - jQuery TagsInput
    • touchTouch.js - touchTouch photo gallery
    • xcharts.js - xCharts plugin for making cool d3-based charts
    • jquery.icheck.js - iCheck plugin for styling checkboxes and radios
  • charts - the following are all initializers for different charts from the theme's pages
    • horizontal_bar_charts.coffee
    • sparkline_samples.coffee
    • xcharts_bar.js
    • xcharts_left_sine.js
    • xcharts_sine.js
  • application.js - the manifest file where all the imports are done - here you can enable or disable the javascript plugins - remember the order is important
  • generic.coffee - initializers for all the components - also defines a Theme object which contains the theme's colors, which are used throughout other initializers
  • calendar.coffee - initializer for fullcalendar
  • custom_actions.coffee - holds theme functionality for opening/closing sidebar buttons, these are mandatory for the theme to work
  • gritter.coffee - gritter module to ease using of the jQuery gritter plugin

HTML files

The html files and partials are organized like this (files that begin with _ are partials - included in other files)

  • layouts
    • layout.erb - general layout file
    • only_topbar.html.erb - layout for login/signup/404 pages
    • _side_navigation.html.erb - contains the sidebars, is rendered in layout
    • _top_navigation.html.erb - top bar menu
  • pages - this folder contains all the pages in the theme demo, structured exactly as the main menu on the left
  • partials - contains different extracted modules/widgets (for simple navigation and DRY purposes)
    • _accordions.html.erb - bootstrap accordion
    • _action_nav_normal.html.erb - the big normal buttons in 3*2 style (dashboard)
    • _action_nav_normal_one_row.html.erb - the big normal buttons in one line
    • _action_nav_normal_rounded.html.erb - the big rounded buttons
    • _activity.html.erb - the activity feed on the dashboard
    • _alerts.html.erb - extracted alert bars
    • _big_chart.html.erb - main chart from the dashboard, with progressbars
    • _bootstrap_editable.html.erb - bootstrap editable sample markup
    • _chat.html.erb - chat layout
    • _closable_chat.html.erb - extracted 'add-a-comment' widget from the activity feed component
    • _data_tables_custom.html.erb - data tables markup
    • _gallery.html.erb - touchTouch gallery
    • _jquery_sliders.html.erb - all the jQuery sliders
    • _modal_form.html.erb - modal with a form
    • _modal_simple.html.erb - modal with simple layout
    • _modal_tables.html.erb - modal with (data) tables
    • _news_with_avatars.html.erb - the news module, with avatars
    • _news_with_icons.html.erb - the news module, with fontawesome icons
    • _sidebar_tabnav_buttons.html.erb - demo for the line-x sidebar buttons
    • _small_chat.html.erb - smaller variant of the chat layout, with a cool send text bar
    • _stats_table.html.erb - a sample table with sparklines and stats
    • _tasks_table.html.erb - a sample table with a task list
    • _widget_boxes.html.erb - the widget boxes with top-right toolbar, from the bottom of the widgets page
